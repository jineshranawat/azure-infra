Orientation

How this course is run

Every session follows the same rhythm: a short theory grounding, a live demo, then a practical you run yourself. Concepts come first so the hands-on part makes sense; code is provided ready to execute, so you spend your time understanding and running, not fighting syntax.

Design choices (kept deliberately approachable)
  • Theory + practical for every topic — Storage, Data Factory, Databricks, SQL and Purview each get their concepts explained, then a hands-on lab. Nothing is "just a look".
  • Databricks + Python are the backbone — PySpark is the workhorse and Delta is the storage format; the other services connect around them.
  • Batch & snapshot — full loads and MERGE upserts, plus Delta time-travel snapshots. Streaming is a short awareness note at the end, so the labs stay reliable.
  • SQL kept practical — read/write Azure SQL from Python; concepts covered, without a deep T-SQL detour.
  • Advanced topics flagged for awareness — change-data-capture, private endpoints, Atlas API automation and multi-stage CI/CD are introduced conceptually so you know they exist, with hands-on left for a follow-on course.

Clean & validate (batch): cast, dedupe, quarantine

Split good rows from bad ones — the FinLedger silver rule.

Databrickseasy
You'll learn
  • Type casting; null handling; the "quarantine bad rows, keep the pipeline running" pattern (like a dead-letter, but in Spark).
notebook — silver splittyped = df.withColumn("amount", F.col("amount").cast("double"))

good = typed.filter(F.col("amount").isNotNull() & (F.col("amount") > 0))
bad  = typed.filter(F.col("amount").isNull()   | (F.col("amount") <= 0))

print("good:", good.count(), "quarantined:", bad.count())
# TXN-20003 (messy row) lands in `bad` — we keep it for review, not lose it

Snapshots & time travel (the safe re-run)

Databrickseasy
You'll learn

notebook — time travel# snapshot historyspark.sql(f"DESCRIBE HISTORY delta.`{silver}`").display()

        .option("versionAsOf", 0)
        .load(silver))

Batch incremental the easy way: MERGE upsert

Load a new day's snapshot and merge it in — no duplicates, no CDC.

Databrickseasy
You'll learn
  • The one incremental pattern we use: MERGE on a key — update if present, insert if new. Re-runnable by design.
notebook — merge upsertfrom delta.tables import DeltaTable

target = DeltaTable.forPath(spark, silver)
updates = good                                   # today's snapshot

(target.alias("t")
   .merge(updates.alias("s"), "t.txn_id = s.txn_id")
   .whenMatchedUpdateAll()
   .whenNotMatchedInsertAll()
   .execute())
# run it twice — row count stays the same. That is idempotency.

Medallion end-to-end in one batch run

bronze → silver → gold, start to finish, one notebook.

Databrickseasy
You'll learn
  • Chain the previous sessions into one runnable pipeline; what belongs in each layer.
notebook — full batch rundef run_medallion(run_date):
    raw   = spark.read.option("header","true").csv(f"{bronze}/loaded/run={run_date}/")
    clean = (raw.withColumn("amount", F.col("amount").cast("double"))
                .filter(F.col("amount") > 0).dropDuplicates(["txn_id"]))
    clean.write.format("delta").mode("overwrite").save(silver)

    gold = clean.groupBy("channel").agg(F.sum("amount").alias("total"))
    gold.write.format("delta").mode("overwrite").save(gold_path)
    return gold.count()

print("gold rows:", run_medallion("session3-lab"))
